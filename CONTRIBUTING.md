# コントリビューションガイドライン

**チームみらい** のインタビューレポートグランプリ用リポジトリへのご協力ありがとうございます。

## 開発言語

このプロジェクトの開発言語は **Python** です。

## 言語に関するルール

このリポジトリでは、以下の内容を **全て日本語** で記載してください：

### 1. コミットメッセージ

コミットメッセージは日本語で記載してください。

**良い例:**
```
新しい機能を追加
バグを修正
ドキュメントを更新
```

**悪い例:**
```
Add new feature
Fix bug
Update documentation
```

### 2. プルリクエスト

- プルリクエストのタイトルは日本語で記載してください
- プルリクエストの説明は日本語で記載してください
- プルリクエストテンプレートに従って情報を記入してください

### 3. コードレビュー

- レビューコメントは日本語で記載してください
- 建設的なフィードバックを心がけてください
- 質問や提案も日本語で行ってください

### 4. Issue

- Issueのタイトルと内容は日本語で記載してください
- Issueテンプレートを使用してください

## 開発フロー

1. このリポジトリをフォークします
2. 新しいブランチを作成します (`git checkout -b feature/新機能名`)
3. 変更をコミットします (`git commit -m '新機能を追加'`)
4. ブランチにプッシュします (`git push origin feature/新機能名`)
5. プルリクエストを作成します（日本語で記述）

## コミット規約

コミットメッセージは以下の形式を推奨します：

- `追加:` - 新しい機能やファイルを追加した場合
- `修正:` - バグやエラーを修正した場合
- `更新:` - 既存の機能を更新した場合
- `削除:` - ファイルや機能を削除した場合
- `リファクタリング:` - コードの構造を改善した場合
- `ドキュメント:` - ドキュメントのみの変更の場合

**例:**
```
追加: ユーザー認証機能を実装
修正: ログイン時のエラーハンドリングを改善
更新: README.mdに使用方法を追記
```

## 質問やサポート

質問がある場合は、Issueを作成してください（日本語で）。
